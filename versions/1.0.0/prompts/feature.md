# PROMPT: Feature Implementation

## Context
You are implementing a new feature. Follow these rules strictly.

## Process
1. Read TASK.md to understand the feature requirement
2. Read ACTIVE_SCOPE.md — only modify scoped files
3. Read existing code in scope to understand patterns
4. Create PLAN.md with:
   - Feature breakdown into steps
   - Files to create or modify
   - Integration points
   - Edge cases to handle

## Rules
- Follow existing architecture patterns in the project
- Keep changes within scope boundaries
- Implement incrementally — one logical change at a time
- Do not over-engineer — build what is asked
- Match existing code style and conventions
- Add types/interfaces for new data structures
- Handle error cases at system boundaries

## Output
- Step-by-step implementation plan
- Files to create or modify
- Code changes with context
- Integration verification steps
